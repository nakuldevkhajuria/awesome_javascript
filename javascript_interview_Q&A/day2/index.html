Question 1: Explain call(), apply() and, bind() methods. Give an example of call(), apply(), bind() 

        All three of the call, bind, and apply methods set the this argument to the function.

        1.  The call and apply methods set this to a function and call the function.
        2.  The bind method will only set this to a function. We will need to separately invoke the function.

    Call

        function test(arg1, arg2){
            console.log(this.num, arg1, arg2); // 100, 10, 20
          }
          
          test.call({num: 100}, 10, 20);
          
          here, {num :100} is an object, which will take place of this, in the function test.
          and Object.num will be accessed 


    Apply


          function test(...arguments){
            console.log(this.num, arguments);//100, [1,2,3]
          }
          
          test.apply({num: 100}, [1,2,3]); 

          The apply method binds the this value to the function and executes the function. It takes the this value and a single array object as parameters, and it returns the value returned by the function, which is called using the apply method.

    bind

        The bind method binds the this value to the function and returns a new function. However, we still need to separately invoke the returned function.

        function test(arg){
         console.log(this.number, arg);
            }

        let bindedFn = test.bind({number: 99}, "argument");

        bindedFn(); // 99, "argument"

Question 3: What are function constructors? 
    The Function() constructor creates a new Function object. Calling the constructor directly can create functions 
    dynamically, but suffers from security and similar (but far less significant) performance issues as eval(). However,
    unlike eval (which may have access to the local scope), the Function constructor creates functions which execute in 
     the global scope only.


    These are the Syntax used by the Functional constructor to create a Function dynamically.

    new Function(functionBody)
    new Function(arg0, functionBody)
    new Function(arg0, arg1, functionBody)
    new Function(arg0, arg1, /* … ,*/ argN, functionBody)

    example:

    const sum = new Function('a', 'b' ,'return a + b');

    console.log(sum(2, 6));
    // expected output: 8

    const multiply = new Function('a', 'b' ,'return a * b');

    console.log(multiply(2, 6));
    // expected output: 8

    const power = new Function('a' ,'return a**3 ');

    console.log(power(2));
    // expected output: 8
Question 4: Explain prototypes? 
        Every object in JavaScript has a built-in property, which is called its prototype. 
        The prototype is itself an object, so the prototype will have its own prototype,
        making what's called a prototype chain. The chain ends when we reach a prototype 
        that has null for its own prototype.
        object --> object constructor --> null
        function --> function constructor --> object constructor --> null
